# CNC Router Configuration

## Overview

This is a **LinuxCNC-based CNC Router** control system configured for a 4-axis machine with dual Y-axis motors. The system uses stepconf-generated configuration files with customizations for advanced features including spindle control, tool changing, and a custom PyVCP interface.

**Machine Name:** CNC_Router  
**Operating System:** LinuxCNC (EMC2)  
**Configuration Version:** 1.1  
**Display:** gmoccapy (modern motion control GUI)

---

## Hardware Configuration

### Machine Axes & Kinematics
- **4-Axis Configuration** (4 motors, 3D travel)
- **Kinematics:** Trivial (coordinates=XYYZ) - 2 motors drive Y-axis simultaneously for balanced gantry movement
- **Work Coordinates:** X, Y, Z motion with dual Y-axis control (JOINT_1 and JOINT_2)

### Axis Specifications

#### X-Axis
- **Range:** -2.0 mm to 500.0 mm
- **Max Velocity:** 30 mm/min
- **Max Acceleration:** 300 mm/s²
- **Step Scale:** 80 steps/mm
- **Home Offset:** -2.0 mm
- **Home Sequence:** 1 (homes first)

#### Y-Axis (Dual Motor)
- **Range:** 0.0 mm to 720.0 mm
- **Max Velocity:** 30 mm/min
- **Max Acceleration:** 200 mm/s²
- **Step Scale:** 80 steps/mm
- **Home Offset:** -2.0 mm
- **Motor Config:** 2 independent stepper motors synchronized (JOINT_1 & JOINT_2)
- **Home Sequence:** -2 (homes together after X)

#### Z-Axis
- **Range:** -150.0 mm to 0.001 mm (downward movement into material)
- **Max Velocity:** 20 mm/min
- **Max Acceleration:** 150 mm/s²
- **Step Scale:** 80 steps/mm

### Spindle Control
- **Max RPM:** 24,000 RPM
- **Min RPM:** 0 RPM
- **Control:** PWM via stepgen
- **Speed Override:** 0.5x - 1.2x
- **Initialization:** Waits for spindle to reach commanded speed before motion

### Hardware Interface
- **Stepper Driver:** Parallel port (LPT1) via HAL parport module
- **Motion Controller:** EMC MOTION module with real-time stepping
- **PWM for Spindle:** Software-generated PWM pulses
- **Real-time Timing:**
  - **Base Period:** 100,000 ns (10 kHz)
  - **Servo Period:** 1,000,000 ns (1 kHz)

## Software Features

### Display & Interface
- **Primary GUI:** gmoccapy - intuitive modern motion control interface
- **Editor:** gedit for text editing
- **Custom PyVCP Panel:** custompanel.xml - custom control interface embedded in main window
- **Embedded Tabs:**
  - **Functions Tab:** GladeVCP-based button interface (button.glade) with toolbuttons
  - **Second User Tab:** Additional controls via vcp_box.glade

### Motion Control
- **Linear Units:** Millimeters (mm)
- **Angular Units:** Degrees
- **Default Feed Rate:** 4.00 mm/min
- **Max Feed Rate:** 30.00 mm/min
- **Feed Rate Override:** 0% - 120%
- **Position Feedback:** Actual (encoder/step position)
- **Position Display:** Relative to current offset

### CNC Programming Features
- **Program Extensions:** 
  - **.nc, .tap** - G-Code files
  - **.png, .gif, .jpg** - Image to G-Code conversion (image-to-gcode filter)
  - **.py** - Python scripts
- **G-Code Path:** G-Code programs loaded from `/home/cnc/Desktop/CNC/`
- **Feed Increments:** 5mm, 1mm, 0.5mm, 0.1mm, 0.05mm, 0.01mm, 0.005mm
- **Grid Snap Options:** 10mm, 20mm, 50mm, 100mm, 1in, 2in, 5in, 10in

### Tool Management
- **Tool Table:** tool.tbl (standard LinuxCNC format)
- **Automatic Tool Changes:** M6 remapped command with custom macro (toolchange.ngc)
- **Tool Change Prolog/Epilog:** Custom sequences for safe tool changing

### Macros & Automation
Located in `macros/` directory:
- **on_abort.ngc** - Emergency abort sequence
- **toolchange.ngc** - Automatic tool change routine
- **zero_all.ngc** - One-command axis zeroing
- **Invoked via:** O<macro_name> call syntax in G-Code

### MDI Commands (Quick Control)
Pre-configured MDI (Manual Data Input) commands for rapid operations:
- **MDI 00:** Zero X-axis: `G10 L20 P0 X0`
- **MDI 01:** Zero Y-axis: `G10 L20 P0 Y0`
- **MDI 02:** Zero Z-axis: `G10 L20 P0 Z0`
- **MDI 03:** Zero All (X, Y, Z): `G10 L20 P0 X0 Y0 Z0`

### Python Integration
- **Python Path:** `/python/` directory
- **Startup Script:** python/toplevel.py
- **Custom Modules:**
  - **remap.py** - Tool change remapping logic
  - **stdglue.py** - Standard LinuxCNC Python support
- **Log Level:** 0 (quiet operation)

## Configuration Files

### Core Configuration
- **CNC_Router.ini** - Main machine configuration (generated by stepconf)
- **CNC_Router.hal** - Hardware abstraction layer (main M3/M4 output, step pulse generation)
- **custom.hal** - User modifications to HAL (preserved during stepconf updates)

### Interface Configuration
- **button.glade** - GladeVCP layout for custom buttons and controls
- **vcp_box.glade** - Second tab interface
- **v_buttonlist.glade & v_buttonlist.hal** - Additional button definitions
- **zero_all_axes.glade** - Specialized axis zeroing interface
- **custompanel.xml** - PyVCP panel configuration

### HAL Post-GUI Configuration
- **postgui_call_list.hal** - HAL commands executed after GUI startup
- **custom_postgui.hal** - User modifications to post-GUI HAL
- **pyvcp_options.hal** - PyVCP signal connections

### Spindle & Motor Control
- **arduino-connector.hal** - Arduino interface (if using Arduino for auxiliary control)

### Runtime Data & Preferences
- **linuxcnc.var & linuxcnc.var.bak** - G-Code variable storage (persistent)
- **rs274ngc.var & rs274ngc.var.bak** - NC interpreter variables
- **CNC_Router.pref** - GUI preferences (colors, window positions, etc.)
- **halshow.preferences** - HAL monitor preferences

### Additional Files
- **pyngcgui.ui** - Python NGC GUI layout
- **autosave.halscope** - HAL scope saved state
